Hook 是什么？

Hook 是一个特殊的函数，它可以让你“钩入” React 的特性。可以使你再不编写class的情况使用state以及其他特性。

什么时候我会用 Hook?

当你再写一个函数组件时，会发现需要向组件内添加一些内置数据，这时候我们就要将这个函数组件，转化成class组件了，现在我们可以有多一种的选择，在不改变本身代码的同时可以使用class组件中state属性，

State Hook

useState在函数组件调用来添加一个”内部”的state,react再重复渲染时，会保留这个state,useState会返回一对值值和一个设置值的函数,当我们在调用时setData函数生成一个新的data时，react会重新渲染并保留它,依照顺序记录每一个state。

  1. 一般来说，一个函数组件，在函数退出后变量就会”消失”，但是 state 中的变量会被 React 保留。
  2. 当我们点击按钮，调用setCount函数时，React会重新渲染这个组件，并把更新的count值传给这个组件
  3. useState()接受一个参数为默认值，该方法返回一个数组，第一个值为定义data的值，第二个为更新data的方法，他们总是成对出现的，
  1. 声明多个useState()

Effect Hook

  • 它与组件的生命周期钩子componentDidMount，componentDidUpdate，componentWillUnmount，具有相同的作用，给函数增加操作副作用的能力
  • 在调用这个hook时，就是告诉React在完成对DOM的更改后运行这个hook,在这里你可以访问到state和props
  • hook 使用了闭包机制，无需js提供了解决方案的时候，引用react的API
  • 与class组件的对比
  • 控制effect hook
  1. 组件在第一次渲染，和更新时都会调用useEffect。
  2. 假如我们所改变的值是一样的，会不会在调用一次useEffect ? 会
  3. 如果你只想调用一次这个effect hook，你可以传递一个[] => componentDidMount
  4. useEffect,第二个参数[var]，当且仅当这个参数发生改变时，才会调用这个effect hook,react会对前一次渲染的[var],与发生改变的[var] 进行比较，如果相等，则会跳过这个effect hook => componentDidUpdate(升级版)
  5. return 一个函数 => componentWillUnmount
  6. 不传 => componentDidMount+componentDidUpdate
  7. 官方说明：会在未来版本中自动添加第二个参数

effect Hook支持使用多个，会按照代码次序依次执行

context Hook

  • useContext()接受一个React.createContext()的返回值
  • useContext()读取 context 的值以及订阅 context 的变化，由上层<MyContext.provider /> 的value props决定
  • 当距离最近的<MyContext.provider /> 更新时，会触发这个hook，重新渲染

Hook 规则

  1. 只在最顶层使用 Hook
    不要在循环，条件或嵌套函数中调用
    保证 Hook 在每一次渲染中都按照同样的顺序被调用
    React 怎么知道当前state是哪个state
